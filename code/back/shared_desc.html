Shared Global State (shared.py) Analysis

1. 시스템 공유 상태 및 세션 플래그
import os

# 공유 상태 변수
current_broadcast_type = None
alert_sent_session = {"fire": False, "reverse": False, "webcam": False}

[설명] 실시간 이벤트 제어의 핵심

current_broadcast_type은 현재 어떤 분석 화면이 송출되고 있는지 추적합니다. 특히 alert_sent_session 딕셔너리는 매우 중요합니다. 한 번 발생한 화재에 대해 알람이 중복으로 울리지 않도록 상태를 잠금(Lock)하는 역할을 수행하여 관제자의 피로도를 줄여줍니다.

2. 실시간 좌표, 영상 정보 및 물리 경로
sim_coords = {"lat": 37.5665, "lng": 126.9780}
current_video_file = {"fire": None, "reverse": None}
latest_frames = {}

# 경로 설정
CAPTURE_DIR = os.path.join(os.getcwd(), "static", "captures")

[설명] 동적 리소스 관리

시뮬레이션의 기준 좌표(서울시청 인근)와 현재 분석 중인 파일 정보를 저장합니다. latest_frames는 AI 분석기가 처리한 가장 최신 프레임을 메모리에 임시 보관하여, 웹 클라이언트가 요청할 때 즉각적으로 응답할 수 있는 버퍼 역할을 합니다. CAPTURE_DIR은 이상 징후 포착 시 이미지를 저장할 절대 경로를 정의합니다.

3. 감지 유형 메타데이터 (Anomaly Data)
ANOMALY_DATA = {
    "fire": {"type": "화재 발생"},
    "reverse": {"type": "역주행"},
    "webcam": {"type": "실시간 현장"}
}

[설명] 시스템 통합 언어 정의

내부 시스템 코드(`fire`, `reverse`)와 사용자에게 보여줄 한국어 명칭을 매핑해둔 사전 데이터입니다. 프로젝트 전체에서 일관된 용어를 사용할 수 있게 해주며, 새로운 분석 유형이 추가될 때 이곳에 정의만 추가하면 되므로 코드의 유지보수성이 극대화됩니다.
