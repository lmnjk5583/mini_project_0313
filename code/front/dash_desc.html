📊 Dashboard.jsx 핵심 분석

1. 실시간 데이터 흐름 (Props의 전달)

Dashboard는 socket, user 등의 정보를 부모(App.jsx)로부터 직접 전달받습니다.

  • socket: 실시간 사고 알림을 받는 무전기.
  • user: 현재 누가 관제 중인지 확인하는 명찰.

데이터가 부모에서 자식으로만 흐르는 '단방향 데이터 흐름'의 전형적인 예시입니다.

2. 복잡한 화면 제어 (useEffect의 마법)

대시보드는 화면이 매우 복잡해서 useEffect를 통해 '자동화된 환경 제어'를 수행합니다.

  • 사고 발생 시: isEmergency가 true가 되면, 지도 패널을 강제로 펼치고 화면에 붉은색 경고(배경 애니메이션)를 띄웁니다.
  • 반응형 처리: 창 크기를 감시(`resize`)하여 모바일(좁은 화면)인지 데스크탑(넓은 화면)인지 실시간으로 판단합니다.
  • 지도 보정: mapRef.current.relayout()을 사용하여 탭 전환이나 창 크기 변화 시 지도가 깨지지 않게 보정합니다.

3. 사용자 정의 훅 (Custom Hook: useAnomalyDetection)

const { isEmergency, ... } = useAnomalyDetection(...)

코드의 가독성을 위해 "이상 탐지"와 관련된 복잡한 로직은 별도 파일(훅)로 빼두었습니다. 대시보드 파일에는 "결과를 어떻게 화면에 보여줄까?"에만 집중할 수 있게 하는 리액트의 고수급 설계 방식입니다.

4. 조건부 렌더링 (보일락 말락)

activeTab === "stats" ? flex : none 같은 코드가 많죠? 이는 탭을 클릭할 때마다 "누구는 보여주고 누구는 숨길지"를 리액트가 판단하게 하는 로직입니다.

특히 지도(`MapPanel`)를 display: none 대신 maxHeight와 opacity를 조절해 스르륵 나타나게 한 것은 '사용자 경험(UX)'을 위해 아주 잘 짠 부분입니다.

5. 백엔드 통신 및 데이터 생명주기

Dashboard는 화면만 그리는 게 아니라 서버와 끊임없이 대화합니다.

  • 초기 데이터 로딩: useEffect(() => { fetchCctvUrl() }, [])를 통해 앱 실행 즉시 CCTV 정보를 확보합니다.
  • 실시간 영상 스트리밍: handleTabChange 호출 시 Date.now()를 쿼리 파라미터로 붙여 캐싱을 방지하고 항상 최신 영상 데이터를 서버에 요청합니다.
  • 사고 조치 보고: resolveAllAlertsAction을 통해 사용자가 조치한 알림 정보를 서버 DB로 전송하여 시스템 상태를 동기화합니다.

이러한 통신 과정은 서버가 보낸 { success: true } 응답을 확인한 후 화면을 갱신하는 '비동기 순차 처리' 구조를 따릅니다.
