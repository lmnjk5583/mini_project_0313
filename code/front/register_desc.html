📝 Register.jsx 상세 코드 가이드

1. 입력값을 객체로 한 번에 관리 (Spread Operator)

const [formData, setFormData] = useState({ name: '', id: '', ... })
`Login`에서는 아이디와 비번을 각각 따로 만들었지만, 여기서는 여러 정보를 하나의 '객체(Object)'에 담았습니다.

setFormData({ ...formData, [e.target.name]: e.target.value })
여기서 ...formData는 "기존의 정보를 그대로 유지하되"라는 뜻입니다. 사용자가 입력한 필드 하나만 쏙 바꿔치기해서 전체 데이터를 최신 상태로 유지하는 기법입니다. (리액트 입문자들이 꼭 배워야 할 패턴!)

2. 서버와의 데이터 주고받기

axios.post(..., formData)
로그인 때는 데이터를 쪼개서 보냈지만, 여기서는 `formData` 전체를 통째로 던집니다. 이렇게 하면 `name`, `id`, `password` 등 모든 회원 정보가 하나의 바구니에 담겨 서버(Flask)로 안전하게 전달됩니다.

3. 깔끔한 사용자 흐름 (UX)

회원가입이 성공하면 곧바로 `Maps('/login')`을 실행합니다. 사용자는 가입 성공 팝업을 보고 나서 바로 로그인 페이지로 이동하게 되어 흐름이 아주 매끄럽습니다.
