📍 useMapMarkers.js 상세 코드 리뷰

1. markersRef : 마커 상태 저장소

const markersRef = useRef({});
[라인 분석] 리액트가 렌더링을 다시 해도 유지되는 마커 객체를 생성합니다.
[의도] 사고 건별로 마커를 개별 관리(`id` 기준)하여 나중에 특정 사고만 삭제할 수 있게 합니다.

2. createMarker() : 지도 오버레이 생성

window.resolveFromMap = (id, type, addr, origin) => resolveEmergency(...); // 전역 함수 연결
const overlay = new window.kakao.maps.CustomOverlay({ ... }); // HTML 기반 마커 생성
markersRef.current[alert.id] = overlay; // 객체에 마커 참조 저장
[라인 분석] 카카오 맵의 CustomOverlay를 사용하여 HTML 기반의 화려한 팝업을 만듭니다.
[핵심] 리액트 내부 함수와 지도상의 HTML 버튼을 연결하기 위해 window.resolveFromMap이라는 전역 함수를 사용했습니다. 이는 외부 스크립트와의 통신을 위한 영리한 트릭입니다.

3. removeMarker() : 마커 정리

if (markersRef.current[alertId]) {
  markersRef.current[alertId].setMap(null); // 지도에서 삭제
  delete markersRef.current[alertId]; // 객체에서 메모리 해제
}
[라인 분석] setMap(null)을 호출해야 카카오 지도 위에서 시각적으로 마커가 사라집니다.
