🔐 Login.jsx 상세 코드 가이드

1. 리액트 입문자를 위한 핵심 문법

입력창의 '실시간 기억' (useState)
onChange={(e) => setUserId(e.target.value)}
사용자가 키보드를 칠 때마다 그 글자를 userId라는 변수에 실시간으로 담습니다. 리액트에서는 화면에 보이는 글자와 실제 변수 값을 이렇게 항상 일치시키는 것이 기본입니다.
페이지 이동 (useNavigate)
navigate('/')
로그인이 성공하면 자동으로 메인 화면(Dashboard)으로 '점프'하게 만드는 도구입니다.

2. App.jsx와의 데이터 연결 (Props)

이 부분이 가장 중요합니다! Login 컴포넌트는 혼자 일하지 않습니다.

  • App.jsx에서 setUser라는 리모컨을 Props(소품/도구)로 전달받았습니다.
  • 로그인 성공 시, 서버에서 받은 유저 정보(res.data.user)를 이 리모컨으로 App.jsx의 상태에 저장합니다.
  • 그 순간, App.jsx가 유저 정보를 인지하고 CCTV 대시보드 화면을 열어주게 됩니다.

3. 서버와의 통신 (Axios & Async/Await)

백엔드 서버(Flask)에 "이 사람 로그인해도 돼?"라고 물어보는 과정입니다.

  • axios.post: 서버에 데이터를 담은 편지를 보냅니다.
  • async / await: 서버의 대답이 올 때까지 잠시 기다렸다가 다음 코드를 실행하라는 뜻입니다. (인터넷 속도에 따라 시간이 걸릴 수 있으니까요!)
  • SweetAlert2: 성공 시 뜨는 "환영합니다" 팝업은 단순한 디자인이 아니라, 사용자에게 "로그인이 확실히 되었다"는 확신을 주는 좋은 UX 장치입니다.

⚠️ 예외 처리 (Error Handling)

아이디가 틀리거나 서버가 꺼져 있을 때, catch 블록이 실행됩니다. 단순히 에러를 내뿜는 게 아니라 setErrorMsg를 통해 화면에 빨간색 경고 문구를 띄워주도록 아주 잘 설계되어 있습니다.
